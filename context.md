### Data Flow
```
User opens browser → SW boots (background) → on_installed fires
  → Load persisted group state from storage::get("groups")
  → Query current tabs via tabs::query
  → Run heuristic grouping (domain match, keyword match)
  → Apply grouping via custom FFI: tabs::group(), tabGroups::update()
  → Persist new state via storage::set("groups", state)

On tab change (on_updated / on_activated):
  → Re-evaluate affected groups
  → Update storage

On popup message (runtime::on_message):
  → Manual regroup, rename, ungroup commands
```

### FFI Surface Required (tabGroups + extras)

**Methods (async, via JsFuture):**
- `chrome.tabGroups.query(queryInfo)` → `Vec<TabGroup>`
- `chrome.tabGroups.update(groupId, updateProps)` → `TabGroup`
- `chrome.tabs.group(options)` → `groupId` (integer)
- `chrome.tabs.ungroup(tabIds)` → `void`

**Events (via Closure + forget):**
- `chrome.tabGroups.onCreated.addListener(cb)` — new group
- `chrome.tabGroups.onUpdated.addListener(cb)` — group changed
- `chrome.tabGroups.onRemoved.addListener(cb)` — group removed

**Optionally:**
- `chrome.tabs.onCreated.addListener(cb)` — new tab (not in v0.2 `#[on]`)

### Constraints & Risks

1. **`#[extension]` macro has no `host_permissions`** — v0.2 only supports `permissions`. Host patterns (if needed) must go in `permissions` array, which Chrome accepts, but it's not the idiomatic MV3 way.
2. **No `tabs::on_created` in v0.2 `#[on]`** — if tab creation events are needed, use raw Closure pattern (same as tabGroups events).
3. **`#[on]` macro is closed** — only the 5 listed events work; you cannot extend it for custom event paths.
4. **Oxichrome v0.2 might not be published yet** — check crates.io. If only `0.1.x` exists, may need to use git dependency or implement APIs from scratch with wasm-bindgen directly.
5. **Closure.forget() causes memory leak** — the closure lives forever (acceptable for SW lifetime). Ensure only one registration per event.
6. **Group state persistence** — `GroupState` must serialize cleanly through `serde_wasm_bindgen`. Avoid recursive types; use flat structs with plain fields.
7. **Heuristic grouping must be deterministic** — same set of tabs should produce same groups, so persisting group IDs across sessions is possible via tab URL matching.
